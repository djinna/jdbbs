What ProdCal Is

A Go web application for book production management, branded as "jdbb studio".

Core Functions
  • Production calendars with Gantt timelines
  • Manuscript transmittal forms
  • Word → PDF/EPUB typesetting pipeline
  • Email notifications (5 types)
  • File tracking & project journals
Tech Stack
  • Go 1.26 + net/http stdlib router
  • SQLite (pure-Go, WAL mode)
  • Embedded static files (single binary)
  • Pandoc + Typst + Python3 for pipeline
  • AgentMail API for email

Book Production Pipeline

1. Upload — POST /api/books/upload

Multipart form, ≤50MB. DOCX stored as SQLite BLOB.

2. Preflight (optional) — POST /api/projects/{id}/preflight

Python script detect-edge-cases.py runs synchronously.
Produces HTML report with severity levels. Cross-references declared custom styles.

3. PDF Conversion — POST /api/books/{id}/convert

Background goroutine (fire-and-forget):

pandoc --from=docx+styles  →  Lua filter  →  .typ source
  → Header patched with spec-driven config
  → typst compile --root / --font-path fonts/
  → PDF stored in DB + book_outputs history table
4. EPUB Generation — POST /api/books/{id}/generate-epub

pandoc docx → epub3 with metadata, TOC, custom CSS from spec.

5. Download — GET /api/books/{id}/download/{pdf|epub}

Serves generated files from DB BLOBs. ⚠️ No auth check.

Real output: "The Twitter Years" DOCX → 8.3MB PDF (most recent: April 14, 2026 01:01 UTC)

Security Assessment

🔴 HIGH — Fix Immediately

  • SHA-256 password hashing — not using bcrypt/scrypt. Vulnerable to rainbow table attacks.
  • /api/books/{id}/download/pdf — NO auth check. Anyone with an ID can download PDFs.
  • /api/projects list — NO auth check. Full project enumeration by unauthenticated users.
  • API key + passwords in git history — SESSION-SUMMARY.txt contains AgentMail key and client passwords.

🟡 MEDIUM — Address Soon

  • Fire-and-forget goroutines for conversion — no timeout, no cancellation, no worker pool.
  • 50MB BLOBs in SQLite — DB will grow fast with book production use.
  • Potential panic in transmittal duplicate — string slice on short key names.
  • EPUB failures not written to DB status — only logged, user sees no error.
  • Preflight runs synchronously — blocks HTTP request for large manuscripts.

🟢 LOW — Note for Later

  • Hardcoded notification recipient (j@djinna.com)
  • Cover images publicly accessible without auth
  • main() doesn't os.Exit(1) on error
  • N+1 query pattern in client digest email
