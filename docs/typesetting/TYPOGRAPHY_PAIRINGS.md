# Print Typography Pairings Guide

## Overview

This guide provides curated typography pairings for book production, inspired by successful approaches like Vellum's streamlined choices. Each pairing includes a text face and heading face designed to work harmoniously together.

## Design Philosophy

- **Limited but Excellent Choices**: Like Vellum, we offer a small set of professional pairings rather than overwhelming options
- **Reader-First**: All choices prioritize readability and book typography best practices
- **Genre-Aware**: Different pairings suit different types of books

## Standard Typography Pairings

### 1. Classic Literary
**Text:** Sabon  
**Headings:** Sabon (Bold/Small Caps)  
**Use for:** Literary fiction, memoirs, poetry  
**Character:** Timeless, elegant, highly readable  
**Notes:** Single-family approach creates cohesive, sophisticated look

### 2. Contemporary Fiction  
**Text:** Minion Pro  
**Headings:** Myriad Pro  
**Use for:** Modern fiction, creative nonfiction  
**Character:** Clean, approachable, professional  
**Notes:** Adobe's superfamily pairing, excellent screen-to-print fidelity

### 3. Scholarly/Academic
**Text:** Crimson Pro  
**Headings:** Source Sans 3  
**Use for:** Academic texts, serious nonfiction  
**Character:** Authoritative, clear hierarchy  
**Notes:** Open-source pairing, strong differentiation between levels

### 4. Business/Self-Help
**Text:** Source Serif 4  
**Headings:** Source Sans 3  
**Use for:** Business books, self-help, how-to guides  
**Character:** Modern, confident, accessible  
**Notes:** Adobe's open-source superfamily, designed to work together

### 5. Technical/Computer
**Text:** Charter  
**Headings:** Fira Sans  
**Code:** JetBrains Mono  
**Use for:** Technical books, programming guides  
**Character:** Readable at small sizes, clear differentiation  
**Notes:** Charter designed for low-res printing, still excellent at high-res

### 6. Historical/Traditional
**Text:** EB Garamond  
**Headings:** EB Garamond (Display weight)  
**Use for:** Historical fiction, classics, traditional subjects  
**Character:** Classical, refined, authentic  
**Notes:** Based on Claude Garamont's specimens

### 7. Friendly/Accessible
**Text:** Alegreya  
**Headings:** Alegreya Sans  
**Use for:** Young adult, light nonfiction, cookbooks  
**Character:** Warm, dynamic, contemporary  
**Notes:** Designed specifically as a book type family

### 8. Modern Neutral (Default)
**Text:** Libertinus Serif  
**Headings:** Libertinus Sans  
**Use for:** General purpose, when no specific mood needed  
**Character:** Neutral, professional, complete  
**Notes:** Our current default, fork of Linux Libertine

## Technical Considerations

### Font Licensing
- All "Standard Typography Pairings" use open-source or freely available fonts
- Premium alternatives require commercial licenses
- We can bundle common commercial fonts or work with client licenses

### Font Features to Enable
- Ligatures (standard + discretionary for literary works)
- Old-style figures for body text
- Lining figures for headings/tables
- True small caps (never fake)
- Kerning and tracking optimized for size

### Size Relationships
Standard sizing ladder (adjustable by trim size):
- Body text: 10.5–11pt (5.5"×8.5" trim)
- Body text: 11–12pt (6"×9" trim)  
- Chapter titles: 2.5–3× body size
- Section heads: 1.5–2× body size
- Subsection heads: 1.2–1.3× body size

## Implementation in Templates

Each pairing comes with:
1. Suggested point sizes for different trim sizes
2. Leading (line-height) recommendations
3. Paragraph spacing values
4. First-line indent measurements
5. Margin proportions

## Streamlined Selection Process

Like Vellum, we present these as themed choices:

1. **Classic** (Sabon)
2. **Modern** (Minion/Myriad)  
3. **Scholarly** (Crimson/Source Sans)
4. **Business** (Source Serif/Sans)
5. **Technical** (Charter/Fira)
6. **Traditional** (EB Garamond)
7. **Friendly** (Alegreya family)
8. **Neutral** (Libertinus family)

Authors simply pick a theme rather than individual fonts.

## Custom Typography Requests

For projects requiring specific fonts:
1. Verify font licensing for print embedding
2. Ensure all necessary weights/styles available
3. Test at actual print size
4. Confirm special character support
5. Check font pairing harmony

## Testing Protocol

Before finalizing any pairing:
1. Print sample pages at actual size
2. Test readability at arm's length
3. Verify special characters render correctly
4. Check font embedding in PDF
5. Ensure EPUB fallbacks work properly
