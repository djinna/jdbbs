Real-Project EPUB-First Test Checklist

Use unique custom style names only: tweet, metadata-p, metadata-c.

Phase 1 — Transmittal Capture

Goal: verify intake fields are correct before export.

Pass criteria
  • All fields persist exactly.
  • No duplicate-style warning appears unless intentionally triggered.

Phase 2 — Pull Into Spec

Goal: verify the transmittal → spec bridge.

Pass criteria
  • Transmittal and spec agree.
  • Type and description survive reload.

Phase 3 — Export Word Template

Goal: verify template generation from the spec.

Pass criteria
  • Export succeeds.
  • All three styles are present and distinguishable.

Phase 4 — Prepare Manuscript in Word

Goal: create a meaningful test DOCX.

Pass criteria
  • DOCX contains both proper custom styles and a small amount of deliberate local formatting noise.

Phase 5 — EPUB-First Run

Goal: verify digital path first.

Pass criteria
  • EPUB builds successfully.
  • Styled content does not collapse into nonsense.
  • Local formatting review path catches what it should.

Phase 6 — Then Print / Typst

Goal: only after EPUB review, verify the print path.

Compact Report Format

Send back results in one of these groups to keep the loop tight:

  • Checkpoint A: transmittal capture: pass/fail; spec pull: pass/fail; surprise: one line
  • Checkpoint B: word template export: pass/fail; styles visible in Word: pass/fail; surprise: one line
  • Checkpoint C: EPUB build: pass/fail; custom-style content survived: pass/fail; stray local formatting caught: pass/fail; surprise: one line
  • Checkpoint D: Typst/PDF build: pass/fail; custom-style content survived: pass/fail; surprise: one line
